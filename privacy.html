Privacy Policy

Home Finance Tracker · Last updated: 6 July 2026

This is a private, personal application. It is operated by an individual (referred to below as “the operator”, “we”, or “us”) solely to track the operator’s own finances. It is not a commercial product and has no other users.

1. Who operates this app

Home Finance Tracker (“the app”) is a local-first personal finance tracker built and run by a single individual on their own device for their own private use. There are no other end users, no accounts to create, and no service offered to the public. 2. What data the app accesses

The app accesses financial information belonging to the operator’s own bank accounts, including:

  • account identifiers (such as IBAN, account name, and account type);
  • account balances;
  • transaction history (dates, amounts, currencies, counterparties, and descriptions).

This data relates only to accounts the operator personally holds and controls. The app does not collect data about any other person and is not designed to be used by anyone other than the operator.

3. How data is accessed and lawful basis

Bank data for supported banks (Openbank, BBVA, and Ibercaja) is retrieved through Enable Banking, a regulated Account Information Service Provider (AISP) operating under the EU PSD2 framework. Data is fetched only after the operator gives explicit consent through their bank’s secure authentication flow. Consent is the lawful basis for this processing, and it can be withdrawn at any time (see section 8).

You can learn more about the AISP at enablebanking.com. Some accounts (such as American Express and El Corte Inglés) are not connected automatically; their data is added only when the operator manually imports a CSV file that they exported themselves.

4. Where data is stored

All financial data is stored locally in a SQLite database on the operator’s own device. It is not uploaded to any cloud service, external database, or server controlled by a third party. The only external service the app communicates with is Enable Banking, and only to fetch the operator’s own account and transaction data.

5. How data is used

Data is used exclusively to display, categorise, and visualise the operator’s personal finances (for example, spending summaries, balances over time, and budgeting views). It is not used for advertising, profiling for third parties, or any commercial purpose.

6. No sharing or selling of data

The operator does not sell, rent, share, or transfer any of this data to third parties. There are no analytics trackers, advertising networks, or data brokers involved. The only outbound data flow is the authenticated request to Enable Banking needed to retrieve the operator’s own data.

7. Data retention

Data remains on the operator’s device for as long as the operator chooses to keep it, so the app can show historical trends. Because storage is local, the operator can delete the local database (or individual records) at any time to permanently remove the data from the device.

8. Revoking bank consent

The operator can revoke Enable Banking / PSD2 access at any time, either through their bank’s online interface or by removing the connection within Enable Banking. Once consent is revoked, the app can no longer fetch new data from that bank. Any data already stored locally can be deleted by the operator as described in section 7.

9. Security

Because the app is local-first, security relies primarily on the security of the operator’s own device. Measures include:

  • data stored only in a local SQLite database on the operator’s machine;
  • Enable Banking API credentials and private keys kept locally and never shared with third parties;
  • encrypted (HTTPS/TLS) connections to Enable Banking;
  • no public server exposing the operator’s financial data.

The operator is responsible for standard device security (screen lock, disk encryption, keeping the machine free of malware, and safeguarding credential files).

10. Your rights

Because the app processes only the operator’s own personal data and stores it locally, the operator retains full control and can access, correct, export, or delete the data at any time directly on the device. Where applicable data-protection law (such as the EU/Spanish GDPR framework) grants additional rights, those rights can be exercised by the operator with respect to their own data.
